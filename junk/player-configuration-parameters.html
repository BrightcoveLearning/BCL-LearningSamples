Player Configuration Parameters

ProductEdition
Video CloudExpress I, Express II, Express III, Professional, Enterprise

You can configure some aspects of the behavior of a Video Cloud player by editing the player's publishing code. This reference topic lists the supported player configuration parameters in Video Cloud, and shows the different syntax used in specifying player configuration parameters in JavaScript, ActionScript, and HTML embed code.

Configuring player parameters can be helpful when you need the same player instance to behave differently in two different places.

In this topic, you will learn about:

  • Syntax for player configuration parameters in JavaScript, ActionScript, and HTML embed code
  • Supported player configuration parameters
  • Using player parameters in URL query strings
  • Ad request player configuration parameters

Syntax for player configuration parameters in JavaScript, ActionScript, and Flash-only Embed code

The configuration parameter names are the same regardless of which type of publishing code you use, but, the syntax varies for JavaScript, ActionScript, and Flash-only Embed code. Read about player publishing codes.

JavaScript publishing code configuration parameters

If you are using JavaScript, you include configuration parameters between the <object> tags in the player publishing code. The JavaScript player configuration parameters take this form:

<param name="param_name" value="param_value" />

ActionScript publishing code configuration parameters

If you are using ActionScript, you include configuration parameters in the init function in the player publishing code. The ActionScript player configuration parameters take this form:

config["param_name"] = "param_value";

Flash-only Embed publishing code configuration parameters

If you are using Flash-only Embed code, you include configuration parameters in the flashVars param in the player publishing code. The HTML embed code player configuration parameters take this form:

<param name="flashVars" value="playerID=13578640001&domain=embed
             &param_name1=param_value1&param_name2=param_value2" />

Supported player configuration parameters

The following table lists the supported player configuration parameters in Video Cloud. Note that all of these parameter names are case-sensitive.

Parameter Type Description
@videoPlayer String The ID or refid of a video to program into the player. See Assigning content to players programmatically.
@playlistTabs String The ID or refid of one or more playlists to program into a player that uses tabs for playlist selection. See Assigning content to players programmatically.
@playlistCombo String The ID or refid of one or more playlists to program into a player that uses a ComboBox (dropdown) for playlist selection. See Assigning content to players programmatically.
@videoList String The ID or refid of a playlist to program into the player. See Designating featured content.
width Number The total width of the player, in pixels.
height Number The total height of the player, in pixels.
autoStart Boolean If true, the video begins to play as soon as the player loads.
bgcolor String A six-digit hexadecimal representation of the player's background color. For example, bgcolor = "#FFFFFF;" (the default) represents white. If you omit this parameter, the player has a white background until it fully loads, which might not be the appearance you want. One suggestion is to match the background color to the color of the page the player loads within.
dynamicStreaming Boolean If true (the default setting), the player can take advantage of Flash Media Server 3.5 and Flash Player 10 smooth streaming capabilities. See Dynamic streaming.
htmlDefaultBitrate Number The default progressive download bitrate to use for HTML5 players. For example, <param name="htmlDefaultBitrate" value="384000" />
htmlFallback Boolean Allows the HTML5 player to be loaded on devices that support Flash but do not have Flash installed. See Allow HTML5 Players in Flash-Supported Environments for more details.
includeAPI Boolean Required to be true in order to use the Smart Player API with this player.
isUI Boolean Required to be true for video players whose templates include UI elements other than the VideoPlayer component, such as a ChromelessVideoPlayer or VideoDisplay and MediaControls components.
isVid Boolean Required to be true for all video players.
labels String The URL of an XML file that describes the label set to use for this player. See Localizing Players for information about how to create a label set file.
language Enum The language to use for the default player labels; one of en (English), fr (French), de (German), it (Italian), jp (Japanese), or es (Spanish). This value overrides the language setting set for the player in the Video Cloud Studio Publishing module.
linkBaseURL String The base URL to use when the viewer clicks the "Share", "Get Link", or "Email" button in the player. Overrides the default linkBaseURL and enables linking to the parent page of the player. See Customizing the link URL.
localizedErrorXML String The URL of a player error message file. This XML file specifies the messages to be displayed in case of player errors. See Localizing player error messages.
majorVersion String The major version number of Flash that is required. By default, Video Cloud players require Flash version 9.0.28 or later. To require Flash 10, use this parameter: <param name="majorVersion" value="10" />.
minorRevision String The minor revision number of Flash 9 that is required. By default, Video Cloud players require Flash version 9.0.28 or later. To require a later Flash 9 version, use a parameter like <param name="minorRevision" value="115" />.
playerId Number The player ID. This parameter is being replaced by the playerKey parameter. If you want to change which player is loaded on a page, you should change the values of both the playerId and the playerKey parameters.
playerKey String A unique identifier for the player, including an encrypted representation of the player ID.
secureConnections Boolean If true, prevents a Flash-based player that is published on a page that uses HTTPS from triggering security warnings. Note that this only prevents security warnings; it does not cause the player to use HTTPS. Also note that you should use this setting only on HTTPS pages along with the secureHTMLConnections parameter; using it on HTTP pages may cause errors. This is only relevant for JavaScript publishing code. See Publishing a Video Cloud Player on an HTTPS Page for more information.
secureHTMLConnections Boolean If true, prevents an HTML5 player that is published on a page that uses HTTPS from triggering security warnings. Note that this only prevents security warnings; it does not cause the player to use HTTPS. Also note that you should use this setting only on HTTPS pages along with the secureConnections parameter; using it on HTTP pages may cause errors. This is only relevant for JavaScript publishing code. See Publishing a Video Cloud Player on an HTTPS Page for more information.
skipContentCall Boolean If true, prevents a content call from being made, if you are using the Flash-only Player API to retrieve content.
showNoContentMessage Boolean If true, enables the display of a warning message if no content has been assigned to the player.
templateErrorHandler String The name of a user-defined JavaScript function to be automatically invoked in response to player error events.
templateLoadHandler String The name of a user-defined JavaScript function to be automatically invoked when the player loads. This provides an entry point for the Smart Player API.
templateReadyHandler String The name of a user-defined JavaScript function to be automatically invoked player ready events.
videoSmoothing Boolean Overrides the default video smoothing behavior for the player; if true (the default value), always applies video smoothing and if false, never applies video smoothing. If absent, the player uses video smoothing.
wmode String Sets the Window Mode property of the SWF file for transparency, layering, positioning, and rendering in the browser. If this attribute is omitted, the default value is window.

Possible values:
  • window - The SWF content plays in its own rectangle ("window") on a web page.
  • direct - Use direct to path rendering. This attribute bypasses compositing in the screen buffer and renders the SWF content directly to the screen.
  • opaque - The SWF content is layered together with other HTML elements on the page. The SWF file is opaque and hides everything layered behind it.
  • transparent - The SWF content is layered together with other HTML elements on the page. The SWF file background color (Stage color) is transparent. HTML elements beneath the SWF file are visible through any transparent areas of the SWF, with alpha blending.
  • gpu - Use additional hardware acceleration on some Internet-connected TVs and mobile devices. In contrast to other wmode values, pixel fidelity for display list graphics is not guaranteed.

Using player parameters in URL query strings

With URL query strings, you can pass player configuration parameters to Video Cloud.

First, copy the publishing code URL from Video Cloud Studio to your clipboard. Notice that this URL is a TinyURL format. You cannot add URL query strings to this format, but you can easily convert it to a long format by pasting the TinyURL into a browser.

The following TinyURL:

http://bcove.me/87tjemnr;

expands into this URL:

http://link.brightcove.com/services/player/bcpid2041213080001?bckey=AQ~~,AAABjnKM-Ck~,C_Yw07pWMKRKOhRLZ7yCOnp00AmK9wBD&bctid=2193277789001

Here are some examples of how you can use URL query strings to customize the player behavior:

Specify an HD video player

The following query string sets the width and height player parameters for a High Definition player size:

http://link.brightcove.com/services/player/bcpid2041213080001?bckey=AQ~~,AAABjnKM-Ck~,C_Yw07pWMKRKOhRLZ7yCOnp00AmK9wBD&bctid=2193277789001&width=1920&height=1080

Publish your video player on a secure page

The following query string uses the secureConnections player parameter for running your player on a secure page. Notice that we have to use the HTTPS protocol with this parameter.

https://link.brightcove.com/services/player/bcpid2144431163001?bckey=AQ~~,AAABjnKM-Ck~,C_Yw07pWMKQ5_jDw69OsKjmHIIu7o21Z&bctid=2193277789001&secureConnections=true

Stop the video from automatically playing

The following query string uses the autoStart player parameter to stop the video from automatically playing when the player loads.

http://link.brightcove.com/services/player/bcpid2144431163001?bckey=AQ~~,AAABjnKM-Ck~,C_Yw07pWMKQ5_jDw69OsKjmHIIu7o21Z&bctid=2193277789001&autoStart=false

Ad request player configuration parameters

In addition, the following player configuration parameters modify a player's ad request behavior. For information about them, see Player configuration parameters for ad operations. Player configuration parameters for a player's ad request behavior include the following:

  • additionalAdTargetingParams
  • adServerURL
  • overrideAds
  • externalAds
  • playlistTargeting
