Tip of the week: If you are maintaining multiple, nearly identical players for different pages, you may be able to use a single player instead by binding component properties to parameter values included in the player publishing code. For example you can include a Label in your BEML template like this:

<Label id="paramLabel" size="18" height="24" text="{player.parameters.VideoCategory}"/>

and then include a parameter in your publishing code to provide the text:

<object id="myExperience734451479001" class="BrightcoveExperience">
  <param name="bgcolor" value="#FFFFFF" />
  <param name="width" value="640" />
  <param name="height" value="360" />
  <param name="playerID" value="852822463001" />
  <param name="playerKey" value="AQ~~,AAAADXdqFgE~,aEKmio9UXajMQo6udNLn584d924n9fJD" />
  <param name="isVid" value="true" />
  <param name="isUI" value="true" />
  <param name="dynamicStreaming" value="true" />
  <param name="VideoCategoy" value="Wildlife Videos" />
  <param name="@videoPlayer" value="734451479001" />
</object>
