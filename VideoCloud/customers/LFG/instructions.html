Full-window all-purpose player

  1. You can use the standard Chromeless Video Player for this, or duplicate it if you want a player that's used only for this purpose
  2. Create an HTML page for this player. You can put other things on the page if you like, but put the player in a <div> tag, like this:
<div class="player-block">

  <!-- put the player publishing code here -->

</div>
  1. Get the player publishing code by going to Media Module, selecting a default video for the player, and using the Quick Publish feature to get the Website code.
  2. Paste the code inside the div tag, and remove or comment out the params for width and height
  3. In your HTML page, also include this CSS code:
<style>
  .player-block {
     width: 100%;
     height: 100%;
     /* the line below is for 16:9 videos; if you have 4:3 videos, it would be padding-bottom: 25%; */
     padding-bottom: 56.25%;
   }
   .BrightCoveExperience {
     width: 100%;
     height: 100%;
   }
</style>
  1. Put this page up somewhere - if you like, it can be "hidden," meaning that you don't have links to it anywhere else - but you need the URL
  2. Now go to the Publishing Module in Video Cloud Studio, select your player, and click Settings to edit the player settings
  3. In the player settings, check the box for Host on your URL, and enter the URL for the page created above
  4. To share any video on this page, go to the Media Module, select the video, select this player in the Quick Publish tool, and copy the URL to send
  5. I'm attaching a simple sample HTML file for the page (with my own player and video, of course)

Upload application

There's a sample app for this here: http://docs.brightcove.com/en/media/samples/create_video.html

A couple of notes on this:

  • I did it all in JavaScript for demo purposes - for production it is risky to expose the write token for your account this way, as it can be use to modify and delete videos in your account by someone who knows what they are doing. It is better to do the processing on the server side, in Java, PHP, or whatever language is available.
  • I included lots of metadata - the description, schedule dates, etc., but you could easily limit this to adding the tags you want.
